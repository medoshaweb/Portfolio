Lab Setup

  • Sensor: Ubuntu VM with Suricata + Filebeat
  • SIEM: ELK stack (Elastic Cloud trial or local Docker)
  • Endpoints: One Windows test VM with Sysmon + Winlogbeat

Detection Scenarios

1) Network Scanning Detection

Detect Nmap SYN scans from attacker subnet.

# Suricata rule example (see download)
alert tcp $EXTERNAL_NET any -> $HOME_NET any (msg:"Nmap SYN Scan"; flags:S; threshold:type both, track by_src, count 10, seconds 5; classtype:network-scan; sid:100001; rev:1;)

2) Suspicious Web Requests

Flag SQLi-like patterns in HTTP URIs.

alert http any any -> $HOME_NET any (msg:"SQLi pattern"; content:"union select"; http_uri; nocase; sid:100002; rev:1;)

3) Endpoint Telemetry (Sysmon)

Detect suspicious process trees (e.g., cmd launching PowerShell with encoded commands).

# Windows Event ID 4688/1 (Sysmon) via Winlogbeat -> SIEM
"process.name: powershell.exe AND event.original: -enc"

4) Incident Response

Capture PCAP, extract IOCs, pivot in SIEM dashboards, and apply blocking (firewall/WAF).

tshark -i eth0 -a duration:120 -w capture.pcap
zeek -r capture.pcap

Dashboards

  • ELK: Create visualizations for Suricata alerts by signature, source IP, and HTTP URI.
  • Windows: Process creations over time and parent-child relationships.

Hardening Tips

  • Implement network segmentation and strict egress filtering.
  • Enable MFA, disable legacy protocols, audit admin groups.
  • Patch cadence and vulnerability management (SLAs).
